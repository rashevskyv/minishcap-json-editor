## 2. Продуктивність та оптимізація

### 2.1. ✅ `calculate_string_width` — Оптимізовано (Speedup 6.7x)

**Файл:** [`utils/utils.py`](file:///d:/git/dev/Picoripi/utils/utils.py)

**Що змінено:**
- Впроваджено **Trie-дерево** для пошуку icon sequences (замість лінійного перебору O(m)).
- Додано **`_WIDTH_CACHE`** для font maps, що уникає перебудови структур при кожному виклику.
- Прямий пошук символів через плоский `dict`.

**Результат:** Час виконання на 52 тис. рядків впав з **573 мс до 84 мс**.

---

### 2.2. ✅ `highlightBlock` — Оптимізовано (Speedup 1.6x - 8x)

**Файл:** [`utils/syntax_highlighter.py`](file:///d:/git/dev/Picoripi/utils/syntax_highlighter.py)

**Що змінено:**
- **Regex Pre-compilation**: Всі 10+ регулярних виразів винесено на рівень модуля/класу.
- **Range setFormat**: Підкреслення глосарію та помилок тепер робиться одним викликом на все слово замість посимвольного циклу.

**Результат:** Швидкість підкреслення глосарію зросла у **8 разів**. Загальний рендеринг блоку став стабільнішим.

---

### 2.3. ✅ `SpellcheckerManager` — Оптимізовано (v0.2.33)
**Файл:** [`core/spellchecker_manager.py`](file:///d:/git/dev/Picoripi/core/spellchecker_manager.py)

**Що змінено:**
- **In-memory Cache**: Додано `self._spell_cache` для миттєвої перевірки вже бачених слів (виключає повторні виклики `spylls`).
- **Persistent Cache**: Кеш результатів зберігається у `spell_cache.json`, що дозволяє пропускати перевірку відомих слів навіть після перезапуску програми.
- **Очистка**: Видалено важке логування (f-strings) з гарячих точок (`is_misspelled`).

**Результат:** Навантаження на CPU при наборі тексту значно знизилося. UI став набагато плавнішим.

---

### 2.4. ✅ `GlossaryManager.find_matches` — Оптимізовано (v0.2.34)
**Файл:** [`core/glossary_manager.py`](file:///d:/git/dev/Picoripi/core/glossary_manager.py)

**Що змінено:**
- Впроваджено **Aho-Corasick** (бібліотека `pyahocorasick`) для пошуку всіх глосарійних термінів за один прохід.
- Оптимізовано `build_occurrence_index` — тепер повний скан проєкту на наявність термінів працює в десятки разів швидше.
- Додано підтримку **багатьох варіантів перекладу** (через `;`) та покращено Bridge Highlighting для багаторядкових термінів.

**Результат:** Перевірка глосарію більше не є вузьким місцем навіть при тисячах термінів.

---

### 2.6. ✅ UI Stability & Recursion Control (v0.2.47)
**Файли:** [`ui/updaters/preview_updater.py`](file:///d:/git/dev/Picoripi/ui/updaters/preview_updater.py), [`components/editor/text_highlight_manager.py`](file:///d:/git/dev/Picoripi/components/editor/text_highlight_manager.py)

**Що змінено:**
- Впроваджено **Reentrancy Guard** (`_in_update_text_views`) для запобігання циклічним викликам оновлення вікон.
- Додано **State Tracking** для підсвітки (перевірка чи змінився блок/позиція перед перемальовкою).
- Винесено важку логіку (перевірка ширини) з `paintEvent`, що розірвало петлю нескінченних перемальовок.
- Виправлено керування прапорцем `is_programmatically_changing_text`, що тепер надійно блокує сигнали Qt під час внутрішніх оновлень.

**Результат:** Повністю усунено `RecursionError` при редагуванні та кліках. Програма стала архітектурно стійкою до каскадних сигналів Qt.
